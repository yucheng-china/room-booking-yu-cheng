Introduction

This is my mini room booking system, first welcome to here! You will see the whole system after enter it and make further actions. Login as a user and you could use the room booking, view your booking details etc. Login as an administrator then you could use the room add, room delete, view customer details etc. The design requirements are listed in the next chapters.


Sample login details

You could login as a user or as an administrator.

Login as user you need (Username+Password+ID), ID is the primary key.

SQL Database Used

There are four tables in my application, they are registration, customer, admin_login and room.

Registration table was used to store the user information (not administrator) which include USERNAME, PASSWORD, ID (primary key), EMAIL. All of them are String type.

SQL: create table registration (username varchar2(20), password varchar2(20), id varchar2(15) primary key, email varchar2(20))


Customer table was store the customer information which include FIRST_NAME, LAST_NAME, EMAIL, PH_NO, ROOM_NO, ID (primary key), CHECK_IN, CHECK_OUT. All of them are String type except room_no.

SQL: create table customer (first_name varchar2(20), last_name varchar2(20), email varchar2(20), ph_no varchar2(20), room_no integer, id varchar2(15) primary key, check_in varchar2(10), check_out varchar2(10))


Admin_login table was store the administrator information which include USERNAME, PASSWORD. All of them are String type.

SQL: create table admin_login (username varchar2(20), password varchar2(20))


room table was store the room information which include ROOM_NO, ROOM_CATEGORY, CAPACITY, BOOKED_BY, STATUS. All of them are String type except room_no.

SQL: create table room (room_no integer, room_category varchar2(20), capacity varchar2(20), booked_by varchar2(20), status varchar2(20))


functionality

The functionality could divided as user functions and administrator functions.

user functions.

Login in and login out.
Make rooms booking.
View their booking details after book.
Query by email.

administrator functions.

Login in and login out.
Insert rooms (room_no should be integer, otherwise will cause view problem in delete)
View room details.
Update room details after inserted (room_no can't be change).
Make room delete after inserted.
View customer details (user who booked rooms already called customer).

Some possible problem

The browse i used in my application was google chrome. I tested it in other browses and the view problem happend eg. The logout, home button show in different position. But all the functions will not be impacted. Therefore, you best run it on goole chrome to have the best view, but it still work in other browses without best view.


As i mentioned in last chapter, you may meet the view problem if you add a room with string or other type of room_no. Because the room_no was created with the integer type. Actually, it should not impact too much, but just avoid it please.


Additional works

Actually there are not too much additionally works, some form-validations i have added in my application were very simple. I don't satisfied too much about it

Login-validation: you should provide Username+Passowrd+ID, each of them are required, you can't miss anyone, and these contents must match the database contents respectively.

Room-Booking-validation: room_no and ID cannot be changed, the check_in and check_out date should be standard form and cannot make booking before 10 April. But i should modify it a little bit so that the check_out date must be the same day as check_in date or several days after the check_in date.

registration-validation: you should provide all information as these information are required.
